## 🚀 快速开始

本章节旨在帮助机器人管理员快速完成插件的部署与初始化配置。

### 1. 插件安装
在 **AstrBot 插件商店** 搜索 `rocom-wiki-astrobot` 并点击安装。

### 2. 数据准备
为确保即搜即得，插件已内置初始数据库。若需手动管理数据，文件路径如下：
* `data/pages.sqlite` (核心数据库)
* `data/pages.jsonl` (备份/增量数据)

### 3. 可视化配置
进入 AstrBot 的 **插件配置页面**，根据需求调整以下核心参数：

| 配置项                  | 推荐设置        | 说明                                               |
| :---------------------- | :-------------- | :------------------------------------------------- |
| `source_mode`           | `auto`          | 插件将根据文件可用性自动切换本地读取或在线爬虫     |
| `update_mode`           | `after_send...` | 优先发送本地缓存，随后异步更新，兼顾速度与数据时效 |
| `crawler_enabled`       | `true`          | 允许插件在本地无数据时通过在线爬虫获取最新内容     |
| `merge_forward_enabled` | `true`          | 建议开启，防止多条查询结果导致群聊刷屏             |

### 4. 激活服务
配置完成后，在 QQ 频道或群聊中对机器人发送：**`/洛克重载`**

## 💬 指令集

> 💡 命令前缀继承 AstrBot 全局设置（如 `/` 或 `#`）。

### 用户指令
* **帮助**：`洛克帮助`
* **查询**：`洛克查 [关键词] (序号)` 或 `洛克查询 [关键词] (序号)`
    * *示例：`洛克查询 小独角兽 1`（直接查看第一个结果）*
    * *注：不带序号时会返回候选列表，回复数字序号即可查看详情。*

### 管理员指令
* **重载配置**：`洛克重载`（刷新本地数据库读取）
* **清理缓存**：`洛克清缓存`

---

## 🔄 进阶数据流

1. **同步更新**：通过 `update_mode` 控制本地数据与 WIKI 的同步策略。
2. **输出策略**：采用结构化展示，包含标题、关键字段、内容摘要及更新时间。
3. **动态补全**：若开启 `after_send_compare_update`，Bot 会先发送本地旧缓存，并在比对到在线更新后自动重发最新内容。
